## 视频相关思考题
### 8.1 虚拟存储的需求背景

1. 寄存器、高速缓存、内存、外存的访问特征？

   1. 访问速度由快到慢
   2. 容量由小到大
   3. 寄存器、高速缓存需要硬件支持

2. 如何理解计算机中的存储层次结构所的理想状态是“均衡繁忙”状态？

   各个介质都在处理繁忙，但是没有构成系统瓶颈。

3. 在你写程序时遇到过内存不够的情况吗？尝试过什么解决方法？

   1. 写数据库时加载B+树遇到内存不足。
   2. 将其写入硬盘。

### 8.2 覆盖和交换

1. 什么是覆盖技术？使用覆盖技术的程序开发者的主要工作是什么？

   将程序划分成不同功能不会同时执行的模块，内存不足时覆盖旧的模块。

2. 什么是交换技术？覆盖与交换有什么不同？

   交换是将进程的地址空间整体的换入换入，而不是像覆盖一样基于功能模块。

3. 覆盖和交换技术在现代计算机系统中还有需要吗？可能用在什么地方？

   交换依然是有必要的，个人计算机中常见的8GB内存实际上常常不够，交换技术可以确保多程序同时运行的体验，覆盖技术就比较没什么用了，编程过于麻烦。

4. 如何分析内核模块间的依赖关系？

   depmod

5. 如何获取内核模块间的函数调用列表？

   1. 静态代码分析
   2. 动态函数追踪


### 8.3 局部性原理

1. 什么是时间局部性、空间局部性和分支局部性？

   1. 时间局部性：如果一块内存在最近被访问了，则在不久之内很可能再度被访问。
   2. 一个程序一段时间内访问的空间通常是某一段相对集中的区域
   3. 分支局部性：空间-时间坐标空间中路径的预期部分只有几个可能的替代方案。这是当指令循环具有简单结构，或者小的条件分支指令系统的可能结果被限制到只有一小部分可能性的情况。

2. 如何提高程序执行时的局部性特征？

   规范程序跳转和数据访问

3. 排序算法的局部性特征？
  * 参考：[九大排序算法再总结](http://blog.csdn.net/xiazdong/article/details/8462393)

### 8.4 虚拟存储概念

1. 什么是虚拟存储？它与覆盖和交换的区别是什么？它有什么好处和挑战？

   只把当前进程需要的部分地址空间载入内存，可以支持大用户空间，且不要求物理地址连续。

2. 虚拟存储需要什么样的支持技术？

   硬件的段（页）式地址转换以及操作系统的置换算法。


### 8.5 虚拟页式存储

 1. 什么是虚拟页式存储？缺页中断处理的功能是什么？

    虚拟页式存储是按照一块固定大小（页）来处理内存，在需要时将数据/程序按页加载到内存，在内存满的时候可以按页把数据换出去。缺页中断处理的是加载内存里不存在的页面。

 1. 为了支持虚拟页式存储的实现，页表项有什么修改？

    驻留位、修改位、访问位、保护位

3. 页式存储和虚拟页式存储的区别是什么？

   能不能支持大于物理地址空间的虚拟空间。

### 8.6 缺页异常

1. 缺页异常的处理流程？
   1. 指令执行中的页表项引用；
   2. 由于页面不在内存，导致缺页异常；
   3. 在外存中查找需要访问的页面备份；
   4. 页面换入；
   5. 页表项修改；
   6. 重新执行导致缺页异常的指令；


2. 虚拟页式存储管理中有效存储访问时间是如何计算的？

   EAT = 访存时间 * (1-p) + 缺页异常处理时间 * 缺页率p

### 虚拟页式存储的地址转换

（3）(spoc) 有一台假想的计算机，页大小（page size）为32 Bytes，支持8KB的虚拟地址空间（virtual address space）,有4KB的物理内存空间（physical memory），采用二级页表，一个页目录项（page directory entry ，PDE）大小为1 Byte,一个页表项（page-table entries
PTEs）大小为1 Byte，1个页目录表大小为32 Bytes，1个页表大小为32 Bytes。页目录基址寄存器（page directory base register，PDBR）保存了页目录表的物理地址（按页对齐）。

PTE格式（8 bit） :
```
  VALID | PFN6 ... PFN0
```
PDE格式（8 bit） :
```
  VALID | PT6 ... PT0
```
其
```
VALID==1表示，表示映射存在；VALID==0表示，表示内存映射不存在（有两种情况：a.对应的物理页帧swap out在硬盘上；b.既没有在内存中，页没有在硬盘上，这时页帧号为0x7F）。
PFN6..0:页帧号或外存中的后备页号
PT6..0:页表的物理基址>>5
```

已经建立好了1个页目录表和8个页表，且页目录表的index为0~7的页目录项分别对应了这8个页表。
